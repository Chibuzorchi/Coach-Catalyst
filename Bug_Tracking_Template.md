## Bug Status Definitions

### Open
- Bug has been reported
- Not yet assigned to developer
- Awaiting triage

### In Progress
- Bug has been assigned
- Developer is working on fix
- Testing in progress

### Resolved
- Bug has been fixed
- Awaiting verification
- Ready for testing

### Closed
- Bug has been verified as fixed
- No longer reproducible
- Testing completed

### Reopened
- Bug was closed but issue persists
- Fix was incomplete
- New information available

## Bug Reporting Best Practices

### Writing Effective Bug Reports
1. **Be Specific**: Include exact steps and expected vs actual results
2. **Be Reproducible**: Ensure others can follow your steps
3. **Be Complete**: Include all relevant information
4. **Be Clear**: Use simple, clear language
5. **Be Objective**: Stick to facts, avoid opinions

### Screenshots and Videos
- Include relevant screenshots
- Highlight the issue area
- Show before/after states
- Record videos for complex issues
- Include error messages and console logs

### Environment Details
- Always include browser and version
- Specify operating system
- Note device type and screen size
- Include network conditions
- Mention any special configurations

## Bug Triage Process

### Initial Triage
1. **Verify**: Confirm bug is reproducible
2. **Categorize**: Assign severity and priority
3. **Assign**: Give to appropriate developer
4. **Estimate**: Provide time estimate for fix
5. **Schedule**: Plan fix in appropriate sprint

### Ongoing Management
1. **Monitor**: Track progress regularly
2. **Update**: Keep status current
3. **Communicate**: Notify stakeholders of changes
4. **Test**: Verify fixes thoroughly
5. **Close**: Mark resolved when confirmed
